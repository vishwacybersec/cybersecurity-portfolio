# Room Name: Web Fundamentals

## Objective

The objective of this room was to understand how websites and web applications work, including HTTP/HTTPS communication, client-server architecture, requests and responses, cookies, sessions, and common web security concepts.

---

## Topics Covered

- How websites work
- Client and Server communication
- HTTP and HTTPS
- Requests and Responses
- GET and POST methods
- Cookies and Sessions
- Status Codes
- DNS basics
- Web application structure
- Introduction to web security testing

---

## Tools Used

- Web Browser
- Browser Developer Tools
- Basic HTTP concepts
- Browser-based lab environment

---

## Key Learnings

I learned how web applications communicate using HTTP requests and responses, how cookies and sessions manage authentication, and why HTTPS is important for secure communication.

I also understood how attackers analyze web applications during reconnaissance and vulnerability assessment.

---

## Commands / Concepts Practiced

- HTTP GET request
- HTTP POST request
- Session handling
- Cookies inspection
- Response status codes
- Basic browser inspection

---

## Practical Understanding

This room helped me understand the foundation of web application penetration testing. It improved my knowledge of how attackers identify weak authentication, insecure sessions, and vulnerable input fields.

This knowledge is directly useful in my SentinelAI project for analyzing web vulnerabilities such as SQL Injection, missing security headers, and security misconfigurations.

---

## Final Thoughts

Web Fundamentals provided strong practical understanding of web application behavior and security concepts, which are essential for penetration testing, vulnerability assessment, and security consulting roles.
